# 原価計算アプリ (genka-app) — 飲食店メニュー汎用版 v4.0

厨房スタッフが直感的に使える、**飲食店の汎用原価管理アプリ** (静的PWA)。どんな料理にも対応できるシンプルな設計で、1皿原価・原価率・粗利額を一目で把握できます。

公開: https://bado1225-oss.github.io/genka-app/

## 設計方針
- 特定メニュー専用ロジックを持たない完全汎用設計
- 入力の手間を最小化(数量+単位だけで原価が出る)
- スマホ現場入力前提のUI
- 1皿原価/原価率/粗利を大きくカード表示、色分けで即判断

## 対象外(意図的に追加しない項目)
- 餃子専用計算・皮・餡・個数等の特定料理専用ロジック
- 外部販売・EC販売 / 発送費 / モール手数料 / 冷凍発送コスト
- OEM費用 / 包材マスター / パック入り原価計算(外販用)

## ☁️ Firebase 同期(任意・複数端末/人で共有)

静的PWA はそのままで、**Firebase Auth + Firestore だけ追加** することで、複数端末・複数スタッフ間でデータ同期できます。未設定時は従来どおり localStorage のみで動作します。

### セットアップ手順
1. [Firebase Console](https://console.firebase.google.com/) でプロジェクト作成
2. **Authentication** → Google サインインを有効化 → 認可ドメインに `bado1225-oss.github.io` を追加
3. **Firestore Database** → 本番モードで作成 → [firestore.rules](firestore.rules) の内容をルールに貼り付け
4. **プロジェクト設定 → 全般 → ウェブアプリを追加** → 表示される `firebaseConfig` オブジェクトをコピー
5. アプリの **設定タブ → Firebase 同期 → Firebase config を設定する** に貼り付け → 保存
6. ページが再読み込みされたら **Google でサインイン**
7. 右上の同期アイコン 🟢 になれば同期中

### 動作仕様
- **自動同期**: 食材・レシピ保存時に Firestore へ自動プッシュ(debounce 1.5秒)
- **リアルタイム反映**: 別端末で更新されたら onSnapshot で自動取得
- **競合解決**: `saved_at` タイムスタンプで新しい方を採用
- **オフライン**: Firestore offline persistence で自動キャッシュ・復帰時同期
- **データ構造**: `users/{uid}/data/main` に ingredients/recipes 1ドキュメントで保存(〜1MB 以内)
- **API Key** はクライアント公開前提。セキュリティは Firestore ルール(認証ユーザーが自分のパスのみ読み書き可)で担保

## v4.0 汎用原価管理に再設計 (2026-04-24)

### レシピ構造(シンプル化)
```
id / name / category(任意) / servings(何人前/皿) / selling_price / target_food_cost_rate / notes
```

### 材料構造(シンプル化)
```
ingredient_id / quantity / unit (g / ml / 個 / 枚 / 本)
```

### 自動計算項目
- 1人前原価 = 材料原価合計 / servings
- 1皿原価 = 材料原価合計
- 原価率 = 1皿原価 / 販売価格
- 粗利額 = 販売価格 - 1皿原価
- 粗利率 = 粗利額 / 販売価格
- 推奨販売価格 = 1皿原価 / 目標原価率

### 表示最優先(画面トップの3大KPI)
- **1皿原価** (大きな金額表示)
- **原価率** (大きな%表示)
- **粗利額** (大きな金額表示)

### 色分けルール(絶対閾値)
- **≤30%: 緑** (cr-low)
- **30〜35%: 黄色** (cr-mid)
- **>35%: 赤** (cr-high)

### 削除した機能(gyoza/店内提供v3.1 → 汎用化で削除)
- 餃子専用モード(1個重量/皮重量/餡原価/皮原価/提供個数別比較)
- メモ機能タブ(端材換算/水抜き計算/皮単価参考/エマルジョン比率)
- 追加コスト入力(仕込み人件費/調理人件費/光熱費/焼成油/タレ等)
- レシピ項目の 歩留まり/kg単価オーバーライドをUIから非表示
  (データモデルには保持、既存データは継続動作)

### 既存データ自動移行
- `per_serving_desc` → `servings`(数字) + `notes`
- `target_cost_ratio` → `target_food_cost_rate`
- items `grams` → `quantity` + `unit='g'`
- 旧 v3.1 フィールド(labor/utility/oil/sauce/piece_weight等)は削除

### 継続機能
- 食材マスタ(野菜27/肉19/魚20/調味料32 = 98件 一括投入)
- 価格履歴 / 仕入先管理 / CSV 書き出し(食材・レシピ・原価結果)
- JSON バックアップ(重複チェック+プレビューimport)
- ダッシュボード: 原価率35%超・粗利高/低TOP5・よく使う食材TOP10・価格更新必要
- スマホ最適化 / スティッキー保存ボタン / PWAオフライン対応

## v3.1 店内提供メニュー特化 (2026-04-24) (参考・削除済)

### 追加コスト(任意・1皿あたり)
材料原価とは別に以下を入力可能。省略可(デフォルト ¥0)。
- 仕込み人件費 / 調理人件費
- 光熱費 / その他製造コスト
- 焼成油原価(餃子等) / タレ原価

### 表示体系の再設計
- **1皿原価** = 材料原価 + 追加コスト (人件費+光熱費+油+タレ) を大きく色分け表示
  - 原価率 <25% = 緑 / 25〜30% = 黄 / 目標超 = 赤
- 販売価格シミュレーターは **1皿原価ベース** で原価率・粗利・推奨販売価格を算出

### 餃子専用モード(店内提供)
外部販売用パック計算を廃止。代わりに:
- **1皿原価の内訳**: 餡原価 / 皮原価 / 焼成油原価 / タレ原価 / その他追加コスト
- **1皿原価(合計)** と 店内販売価格 / 粗利額 / 原価率 を並列表示
- **店内提供個数別比較** (5個/6個/10個提供): 食材原価 + 1皿原価 + 粗利 + 原価率を自動計算
  - 油・タレ・人件費・光熱費は1皿固定、食材原価のみ個数に比例

### ダッシュボード改善
- 登録メニュー / 平均原価率 / **原価率30%以上のメニュー数**
- 💰 粗利額が高いメニュー TOP5
- 📉 **粗利額が低いメニュー TOP5** (要改善メニュー)
- 📋 **よく使う食材 TOP10** (レシピ参照回数で集計)
- ⏰ 価格更新が必要な食材(30日以上)

## v3.0 機能一覧

### 1. レシピ原価の詳細表示
- 1人前原価 / 1個あたり原価(per_serving_descから個数自動抽出)
- 仕込み総原価 / 材料別原価 / 材料別構成比
- 歩留まり反映後の実使用原価
- ロス込み原価(ロス率×仕入原価)

### 2. 販売価格シミュレーター
各レシピに販売価格・目標原価率・目標粗利を設定すると、以下を自動計算:
- 原価率 = 原価 / 販売価格
- 粗利額 = 販売価格 - 原価
- 粗利率 = 粗利額 / 販売価格
- 推奨販売価格(原価率ベース) = 原価 / 目標原価率
- 推奨販売価格(粗利ベース) = 原価 + 目標粗利
- 目標原価率を超えると赤系警告 + 推奨価格提案

### 3. 食材マスタ(強化)
各食材に以下を保持:
- `supplier_name` 仕入先
- `price_type` 市場/仕入れ/都度入力/固定価格
- `standard_price` 基準価格
- `actual_purchase_price` 実仕入価格
- `unit` 購入単位 (kg/L/個/枚/本/袋)
- `yield_rate` 標準歩留まり
- `loss_rate` 搬送・保管ロス率
- `last_updated` 価格最終更新日時
- `is_variable_price` 変動大フラグ
- `price_history` 価格変更履歴(自動記録)

### 4. 価格履歴機能
食材編集時に実仕入価格を変えると `{date, from, to, supplier, memo}` が自動で履歴に追加されます。モーダル下部に最新10件を表示、過去の価格上昇傾向を把握できます。

### 5. 仕込みロット計算
レシピ編集下部の「仕込みロット計算」で、人前数を入れると:
- 仕込み総重量 / ロス込み必要量
- 仕込み総原価 / ロス込み原価
- 材料別の必要量(使用g・仕入g・ロス込みg)が一覧表で出力

### 6. 餃子専用モード
レシピカテゴリ=餃子 のとき自動表示。
- 1個重量 / 1個あたり皮重量 → 1個あたり餡量 自動算出
- 餡総量 / 皮必要枚数 / 皮単価/個
- 炭火焼鶏使用量 / 豚ミンチ使用量
- 野菜水抜き前後重量 / 塩量
- 完成個数 / 1個原価 / 5個入り原価 / 10個入り原価
- パック数別(5/6/8/10/12個入り)の原価・粗利・原価率早見表

### 7. ダッシュボード改善
トップに以下を表示:
- 登録レシピ数 / 食材マスタ数
- 平均原価率 / 価格未更新数 / 都度入力数
- 📈 原価率が高いレシピ TOP5
- 💰 粗利額が高いレシピ TOP5
- ⏰ 価格未更新の食材(30日以上)
- 🟠 都度入力が必要な食材

各項目はクリックで該当レシピ/食材編集に直遷移。

### 8. CSV/JSON入出力(強化)
設定タブから:
- JSON 全データ書き出し(食材+レシピ+履歴)
- JSON 読み込み(**重複チェック + プレビュー** → 既存追加/上書きを選択)
- CSV 食材マスタ書き出し(UTF-8 BOM付・Excel対応)
- CSV レシピ書き出し
- CSV 原価計算結果書き出し(原価率・粗利・推奨価格含む)

### 9. UI改善
- 入力フォーム・数字入力を大きめに
- 保存ボタンをスティッキー(画面下固定)
- 原価・粗利・原価率はカード表示
- 目標原価率を超えたレシピ/原価率が高い値は **赤系表示**
- 価格未更新30日以上の食材は黄色ハイライト

### 10. データ構造の整理
- 既存 v2 データは起動時に自動マイグレート
- 新フィールドは既存食材に補完(kg_price → standard_price/actual_purchase_price 同期)
- localStorage `genka_v2` キーは維持(バックアップ互換)

## 既存機能(継続)

- 食材マスタ(野菜27+肉19+魚20+調味料32=98件)一括投入
- 野菜マスター / 肉マスター / 魚マスター / 調味料マスター の TS/JSON/CSV
- カテゴリ→食材 2段階選択
- レシピ内での kg単価オーバーライド(このレシピだけ)
- 歩留まり(%)調整
- メモ機能タブ: 端材換算・水抜き計算・皮単価参考・エマルジョン比率

## 使い方

1. 公開版を開く
2. 「設定」タブ → 野菜/肉/魚/調味料マスター を一括投入
3. レシピタブで「新規レシピ」→ カテゴリ=餃子 を選択
4. 材料行で「カテゴリ → 食材」を選び、使用g・歩留まりを入力
5. 販売価格シミュレーターに販売価格を入力 → 原価率・粗利を確認
6. 仕込みロット計算で人前数を指定 → 必要仕入量を把握
7. 餃子モードで1個重量・皮重量を調整 → パック別原価を確認
8. 保存(画面下のスティッキーボタン) / データは端末のブラウザに保存

## ローカル実行

```bash
cd genka-app
python3 -m http.server 8779
```

http://localhost:8779 を開く。
